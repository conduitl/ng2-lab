Important note from angular docs about side effects. The expression could call something like getFoo(). Only we know what getFoo() does. If getFoo() changes something an we happen to be binding to that something, we risk an unpleasant experience. Angular may or may no display the changed value. Angular may detect the change and throw a warnin error. Our general advice: stick to data properties and to methods that return values and do no more.

The brackets []

The brackets tell Angular to evaluate the template expression. If it is omitted, Angula treats the string as a constant and initializes the target property with that string.

Omit brackets [] when all of the following are true:

  • The target property accepts a string value.
  • The string is a fixed value that we can back into the template.
  • This initial value never changes.
